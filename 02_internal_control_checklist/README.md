# 02 内部統制チェックリスト

## 概要

内部統制の確認項目をSQLiteデータベースで管理するツール。チェック項目の追加・ステータス更新・削除が行え、進捗率をプログレスバーでリアルタイム表示する。

## 起動

```powershell
cd 02_internal_control_checklist
python gui.py
```

初回起動時に `data/checklist.db` が自動作成され、`data/checklist.csv` のサンプルデータが投入される。

## 使い方

1. 起動するとサンプルチェック項目が一覧表示される
2. 項目を選択して「ステータス更新」でステータス・コメントを変更
3. 「項目追加」で新規チェック項目を登録
4. 進捗バーが自動更新される（「完了」「該当なし」が分母から除外されない集計）
5. 「CSV出力」で `results/checklist_report.csv` に全データを出力

## 行の色分けルール

| ステータス | 表示 |
|-----------|------|
| 完了 | グレー文字 |
| 該当なし | 薄いグレー文字 |
| 進行中 | 青文字 |
| 未着手（優先度：高） | 黄色背景 |

## ファイル構成

```
02_internal_control_checklist/
├── main.py          # SQLite CRUD・進捗計算ロジック
├── gui.py           # Tkinter GUI（Treeview + ProgressBar + ダイアログ）
├── data/
│   ├── checklist.csv    # 初期投入用サンプルデータ（12項目）
│   └── checklist.db     # SQLite DB（初回自動生成）
└── results/
    └── checklist_report.csv   # CSV出力
```

## サンプルデータのカテゴリ

- 財務統制（売上承認・月次照合・現金実査）
- 購買統制（権限規程・相見積もり・検収照合）
- IT統制（アクセス権・パスワードポリシー・バックアップ）
- 人事統制（採用審査・評価フロー・退職者対応）

## 主要ロジック（main.py）

| 関数 | 内容 |
|------|------|
| `init_db()` | DB初期化・サンプルデータ投入 |
| `load_all()` | 全チェック項目を辞書リストで返す |
| `add_item(...)` | 新規項目をINSERT |
| `update_status(id, status, comment)` | ステータス・コメントをUPDATE |
| `delete_item(id)` | 項目をDELETE |
| `get_progress()` | 総数・完了数・進捗率を返す |

## 面接でのアピールポイント

- **監査経験そのまま：** J-SOXや内部監査で使うチェックリストをツール化した実体験として語れる
- `sqlite3`（Python標準ライブラリ）でデータ永続化を実装したことを示せる
- Toplevel ダイアログを使ったモーダルウィンドウ設計を説明できる
- 「紙やExcelで管理していたものをDBに移行した」という業務改善ストーリーが作れる
