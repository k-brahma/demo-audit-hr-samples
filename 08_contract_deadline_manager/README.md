# 08 契約書 期限管理ツール

## 概要

契約書の終了日をSQLiteで管理し、今日からの残日数を自動計算して期限の近い契約を色分け表示するツール。期限管理の見落としを防ぐことを目的としている。

## 起動

```powershell
cd 08_contract_deadline_manager
python gui.py
```

初回起動時に `data/contracts.db` が自動作成され、6件のサンプル契約が登録される。

## 使い方

1. 起動するとサンプル契約一覧が表示される（残日数の近い順）
2. 「➕ 契約追加」で新規契約を登録
3. 契約を選択して「🗑 削除」で削除
4. 「🔄 更新」で今日付けの残日数に再計算
5. 「💾 CSV出力」で `results/contracts_report.csv` に出力

## 行の色分けルール

| 残日数 | 緊急度 | 背景色 |
|--------|--------|--------|
| 期限超過 | 期限切れ 🔴 | 薄い赤 |
| 30日以内 | 危険 🔴 | 薄い赤 |
| 31〜90日 | 警告 🟡 | 薄い黄 |
| 91日以上 | 正常 🟢 | 薄い緑 |

タイトルバーに「危険○件・警告○件」のカウントが常時表示される。

## ファイル構成

```
08_contract_deadline_manager/
├── main.py          # SQLite CRUD・残日数計算・緊急度判定ロジック
├── gui.py           # Tkinter GUI（色分けTreeview + 追加ダイアログ）
├── data/
│   └── contracts.db     # SQLite DB（初回自動生成・サンプル6件）
└── results/
    └── contracts_report.csv   # CSV出力
```

## 登録データの仕様

| フィールド | 内容 | 形式 |
|-----------|------|------|
| 契約名 | 契約書の名称 | テキスト |
| 相手先 | 取引先名 | テキスト |
| 開始日 | 契約開始日 | YYYY-MM-DD |
| 終了日 | 契約終了日 | YYYY-MM-DD |
| カテゴリ | 業務委託・売買・賃貸借・保守・ライセンス・その他 | 選択式 |
| 備考 | 自由記述 | テキスト |

## 主要ロジック（main.py）

| 関数 | 内容 |
|------|------|
| `init_db()` | DB初期化・サンプルデータ自動投入 |
| `load_all()` | 全契約を残日数・緊急度付きで返す |
| `add_contract(...)` | 新規契約をINSERT |
| `delete_contract(id)` | 契約をDELETE |
| `export_results()` | CSVエクスポート |

## 面接でのアピールポイント

- **総務・法務の実務課題：** 「更新失念で損害が発生するリスク」という現実の問題解決として語れる
- SQLiteによるデータ永続化（アプリを閉じてもデータが残る）の実装を説明できる
- タイトルバーを動的に更新してアラート件数を常時表示するUIの工夫を示せる
- 入力バリデーション（日付フォーマットチェック）の実装を説明できる
